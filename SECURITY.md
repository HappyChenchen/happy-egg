# 安全策略

## 支持范围

当前项目处于 Beta 阶段，只维护 `main` 分支的最新版本。旧构建不会单独获得安全补丁。

## 报告漏洞

请使用 GitHub 的 [Private vulnerability reporting](https://github.com/HappyChenchen/happy-egg/security/advisories/new) 私密提交安全问题。

请不要在公开 Issue、讨论区或社交媒体披露尚未修复的漏洞。报告中请包含：

- 受影响版本或提交
- 可复现步骤
- 实际影响
- 建议修复方式（如有）

## 当前安全边界

- 客户端与公网 relay 通过 TLS WebSocket 通信。
- Relay 校验消息类型、名字长度、配对码、稳定 ID、动作名和素材名。
- 每个配对房间最多两个连接；未完成邀请默认 10 分钟过期。
- 每个连接每分钟最多发送 20 条受限消息。
- 长期互动要求双方都上报彼此为好友，并且接收方在线。
- Relay 容器使用非 root 用户、只读文件系统和 `no-new-privileges`。

## 已知限制

- 当前没有用户账户、登录、设备证明或服务端持久化好友关系。
- 稳定 ID 由客户端生成和保存，不是密码学认证身份。
- 8 位配对码适合临时邀请，不应作为长期秘密。
- 互动内容由 relay 解密并转发，不是端到端加密。
- Ad-hoc 签名只用于本地构建，不能替代 Apple Developer 签名与公证。

## 数据处理

客户端本地保存：

- 宠物名字和显示设置
- 32 位稳定宠物 ID
- 好友名字、稳定 ID 和历史配对房间 ID

Relay 进程内存保存：

- 当前 WebSocket 连接
- 未完成的临时配对房间
- 当前在线身份和好友订阅
- 用于频率限制的短期时间戳

Relay 当前不使用数据库，也不持久化互动消息。进程重启后上述内存状态会清空。
