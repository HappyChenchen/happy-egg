# 安全策略

## 支持范围

当前项目处于 Beta 阶段，只维护 `main` 分支的最新版本。旧构建不会单独获得安全补丁。

## 报告漏洞

请使用 GitHub 的 [Private vulnerability reporting](https://github.com/HappyChenchen/happy-egg/security/advisories/new) 私密提交安全问题。

请不要在公开 Issue、讨论区或社交媒体披露尚未修复的漏洞。报告中请包含：

- 受影响版本或提交
- 可复现步骤
- 实际影响
- 建议修复方式（如有）

## 当前安全边界

- 客户端与公网 relay 通过 TLS WebSocket 通信。
- Relay 校验消息类型、名字长度、宠物号、稳定 ID、认证令牌、动作名和素材名。
- 单个 WebSocket 消息帧上限为 16 KiB，超限连接在进入 JSON/正文处理前关闭。
- 永久宠物号只能用于发送申请；建立好友关系需要接收方明确接受。
- 稳定 ID 首次注册后绑定 64 位随机设备令牌，Relay 仅保存令牌的 SHA-256 哈希。
- 每个连接每分钟最多发送 20 条受限消息；好友留言按认证稳定 ID 跨并行连接/重连限制为每分钟 30 条，并按来源地址限制为每分钟 300 条。来源地址默认取 TCP 对端；只有受控反向代理部署显式开启 `MACPET_TRUST_PROXY` 后才读取代理转发头。
- 在线身份注册按稳定 ID 跨重连限制为每分钟 21 次、按来源地址限制为每分钟 600 次；同一来源地址每小时最多创建 20 个新身份，注册表总容量为 100000 个身份。
- 长期互动要求双方客户端订阅彼此、服务端关系仍有效，并且接收方在线。
- 好友留言（短文字或预设贴纸）要求两个已注册身份的服务端关系仍为双向；删除好友由 Relay 确认后同时撤销双方关系并留下墓碑，旧会话不能用过期好友列表恢复权限。
- 好友关系每个身份最多 100 个；删除请求按稳定 ID 每分钟 10 次、按来源地址每分钟 100 次限流，且只有真实双向关系或已有墓碑的幂等重试会被接受。
- 好友申请及结果全局最多持久化 10000 条；未处理申请 30 天过期，已接受/拒绝但未 ACK 的结果 7 天过期，均由每分钟清理器主动删除。
- 好友申请与留言 ACK 只接受已认证会话，两类 ACK 共用限流：按稳定 ID 每分钟 300 次、按来源地址每分钟 3000 次；未认证尝试不会占用其自报稳定 ID 的额度。
- 文本长度上限 300 个 Unicode code point，贴纸走服务端白名单；接收方确认本地保存后 Relay 立即删除服务端副本。
- 未确认留言使用 7 天 TTL，并由每分钟清理器主动删除（最迟约 7 天加 1 分钟）；每个接收方最多 50 条，全局硬上限 5000 条，达到全局上限时拒绝新留言而不淘汰其他接收方的数据。
- Relay 容器使用非 root 用户、只读根文件系统、独立数据卷和 `no-new-privileges`。

## 已知限制

- 当前没有用户账户、登录或设备证明；Relay 持久化的是接受申请、兼容迁移的双向好友声明及删除墓碑，并不是可找回、可审计的账户级好友系统。
- 设备令牌是本机持有的随机凭据，但没有找回、跨设备迁移或撤销登录机制。
- 6 位宠物号是公开标识而不是秘密；公开发布可能收到陌生申请，但不能让陌生人直接发送互动。
- 互动内容由 relay 解密并转发，不是端到端加密。
- 发给好友的留言会以明文存放在 Relay 数据卷中，直到对方确认保存，或 7 天 TTL 到期后的下一次每分钟清理；Relay 管理员在此期间可读取留言内容。
- Ad-hoc 签名只用于本地构建，不能替代 Apple Developer 签名与公证。

## 数据处理

客户端本地保存：

- 宠物名字和显示设置
- 32 位稳定宠物 ID
- 64 位设备认证令牌和当前宠物号缓存
- 好友名字和稳定 ID
- 最近收到的好友留言（Mac 与网页端各自最多 50 条；Mac 含已读状态）

Relay 数据卷持久化：

- 稳定 ID、宠物名字、宠物号、设备令牌哈希、服务端好友 ID 和删除关系墓碑
- 待处理好友申请，以及尚未被双方确认保存的申请结果（全局最多 10000 条；待处理 30 天、结果 7 天 TTL，每分钟主动清理）
- 尚未确认保存的好友留言（7 天 TTL、每分钟主动清理；每个接收方最多 50 条，超出丢弃该接收方最旧一条；全局最多 5000 条）

Relay 进程内存保存：

- 当前 WebSocket 连接
- 当前在线身份和好友订阅
- 用于按认证稳定 ID 和来源地址限制注册、删除与留言的短期时间戳

Relay 使用本地 JSON 注册表而不是外部数据库。实时互动（拍一拍、爱心、庆祝）不持久化；好友留言在接收方确认本地保存前、且未被 7 天 TTL 清理时保存在注册表中。进程重启后在线连接和内存限流窗口会清空，注册表内容（含未确认留言与删除墓碑）会从 Docker volume 恢复。旧版客户端的临时配对房间仍只存在于内存中。
