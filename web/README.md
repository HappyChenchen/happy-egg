# 网页联动台

网页联动台是一个无需构建的静态调试工具，使用与 macOS 客户端相同的永久宠物号、好友申请、在线状态、互动和留言协议。它不是 macOS 客户端的完整替代品。

## 启动

在仓库根目录运行：

```sh
make web
```

打开 [http://localhost:4173/web/](http://localhost:4173/web/)。

## 使用

1. 网页连接后会显示自己的 6 位宠物号，点击号码可复制。
2. 在“操作”菜单输入朋友的宠物号并发送申请，或在菜单中接受朋友发来的申请。
3. 好友在线后可发送拍一拍、爱心和庆祝。
4. 在菜单的“留言与贴纸”里可给好友发短文字或预设贴纸；好友离线也能发送，会在其上线后送达。“消息记录”保留最近 50 条收到的内容。
5. 网页右上角按钮或宠物右键菜单可调整大小、隐藏宠物、删除好友或断开连接。

默认连接 `wss://happypuppy.io/ws`。本地 relay 可通过查询参数指定：

```text
http://localhost:4173/web/?relay=ws://localhost:8080/ws
```

## 自动双端验收

本机安装 Python Playwright 与 Google Chrome 后，可在仓库根目录运行：

```sh
make test-e2e
```

该命令使用两个隔离的 Chromium BrowserContext 和动态本地端口，真实完成建友、双向文字、贴纸、离线排队、Relay 换 PID 重启持久化、重连补发、重复投递恢复存储、50 条多标签页合并边界、保存后 ACK 清队列、接收端刷新后的本地记录恢复，以及离线一方重连后的删除好友对账，并继续运行 Relay 全量测试、JavaScript 语法检查及 `swift build`。结构化结果、注册表和阶段截图写入被忽略的 `work/` 目录。

## 限制

- 网页身份、设备令牌、好友列表和最近 50 条收到的留言只保存在当前浏览器的 `localStorage`；清除网站数据会创建新的宠物身份并删除本地消息记录。
- 网页目前一次显示一个最近添加或互动的好友。
- 这是手动调试工具，目前没有独立生产部署流程。
